コスト配分

テストによる品質担保とスピードのトレードオフ

- 継続的な更新を行うアプリケーションの場合に変更による不具合発生が検知できるので更新のスピードを出せるはず
- テスト実装で時間がかかるのはテスト粒度設計の問題と考える
    - 静的解析は早めに入れておいて損はない
        - 後から入れると修正対応で地獄
    - 結合テストをメインに据えるとコスト対効果でバランスの良い
        - [The Testing Trophy](https://kentcdodds.com/blog/write-tests)
